1.1. A tag "i" no HTML é usada para indicar que o texto contido dentro dela é um texto com ênfase ou destaque em relação ao texto em torno dele. Anteriormente, essa tag era comumente usada para itálico, mas com a evolução dos padrões web, seu uso para itálico foi substituído pela tag "em".

Portanto, atualmente, a tag i é mais frequentemente usada para fins semânticos, como destacar um termo técnico, uma palavra em um idioma estrangeiro ou uma frase de um texto, sem implicar que o texto deve ser exibido em itálico. O estilo visual, incluindo a itálico, deve ser aplicado usando CSS, separando a estrutura do documento da sua apresentação visual.

Aqui está um exemplo simples para ilustrar a diferença entre i e em:

Isso é um texto com ênfase.

Isso é um texto com ênfase.

Neste exemplo, ambos os i e em podem ser exibidos como itálico por padrão no navegador, mas semanticamente eles têm significados diferentes. O i é mais genérico e pode ser usado para texto com destaque visual, enquanto o em é usado para ênfase semântica, indicando que a palavra é enfatizada no contexto da frase.


1.2. A tag strong e a tag b no HTML são usadas para destacar texto, mas têm propósitos diferentes.

A tag b é usada para estilizar texto de forma que ele apareça em negrito. Ela não adiciona significado semântico ao conteúdo, ou seja, não informa aos mecanismos de busca ou leitores de tela que o texto é mais importante do que o restante do conteúdo.

A tag strong, por outro lado, é usada para dar ênfase forte ao texto, indicando que o conteúdo é de maior importância ou urgência. Além de estilizar o texto em negrito (como b), a tag strong comunica o significado semântico de ênfase forte, o que pode ser relevante para acessibilidade e SEO.

Aqui está um exemplo para ilustrar a diferença:

Esse é um texto em negrito sem significado semântico.

Esse é um texto com ênfase forte, indicando maior importância.

Neste exemplo, visualmente ambos os textos podem aparecer em negrito, mas strong carrega um peso semântico adicional que b não possui.


1.3. No HTML5, a tag u um significado semântico ligeiramente diferente. Em vez de simplesmente sublinhar texto para efeito estético, é recomendado usar a tag u para marcar texto que tenha um estilo diferente por razões semânticas específicas, como para indicar o nome de uma obra ou para marcar um termo em um contexto específico.

Para sublinhar texto com CSS, você pode usar a propriedade text-decoration. Este método é mais flexível e semântico, permitindo a separação de estilo e conteúdo. Exemplo abaixo:

Este texto está sublinhado.

Usar a tag u apenas para estilização visual pode ser confuso para usuários que dependem de tecnologias assistivas. Para garantir acessibilidade, é melhor usar o CSS para sublinhar texto e reservar a tag u para casos onde sublinhar o texto tenha significado semântico específico.

Embora a tag u ainda seja válida em HTML, seu uso para estilização visual não é recomendado. Em vez disso, deve-se usar CSS para sublinhar texto e reservar a tag u para casos onde o sublinhado tenha um propósito semântico específico. Isso resulta em um código mais limpo, semântico e acessível.

1.4. A tag br no HTML é utilizada para inserir uma quebra de linha no texto. Quando o navegador encontra essa tag, ele interrompe a linha atual e continua o texto na linha seguinte. A tag br é um elemento vazio, o que significa que não possui uma tag de fechamento.

Embora br seja útil para quebras de linha simples, é importante usar essa tag de forma apropriada e evitar abusos. Em muitos casos, usar CSS para controlar o layout e espaçamento do texto pode ser uma abordagem mais semântica e flexível. Por exemplo, você pode usar margens ou preenchimento em CSS para criar espaçamento vertical em vez de múltiplas quebras de linha.

Neste exemplo, a classe endereco usa a propriedade line-height para ajustar o espaçamento entre linhas, melhorando a legibilidade sem depender de múltiplas tags br.

4. Listas Ordenadas e Não Ordenadas

(voltar)

4.1. Listas Ordenadas

Uma lista ordenada possui numeração para cada item da lista. Utilizando a tag "ol" faço uma lista ordenada, para adicionar os itens nessa lista eu coloco a tag "li" dentro da "ol". Conforme o exemplo abaixo:
  1. Paula
  2. Alessandra
  3. Rodrigues
  4. dos Santos
É possível colocar uma lista ordenada dentro de uma "li". Conforme abaixo:

  1. item 1
  2. item 2
  3. item 3
  4. item 4
    1. item 4.1
    2. item 4.2
    3. item 4.3

4.2. Listas Não Ordenadas

Uma lista não ordenada não possui numeração para cada item da lista e sim marcadores "bolinhas" ou outro tipo de marcador que desejar. Utilizando a tag "ul" faço uma lista não ordenada, para adicionar os itens nessa lista eu coloco a tag "li" dentro da "ul". Conforme o exemplo abaixo:
  • item 1
  • item 2
  • item 3
